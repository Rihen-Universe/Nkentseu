## 1. Vue d'ensemble du Launcher

### Description Générale

Le Launcher est le **point d'entrée de l'IDE**, affiché avant l'ouverture de l'éditeur principal. C'est une fenêtre indépendante (non liée à l'éditeur), avec une interface entièrement custom — pas de file dialog système natif, pas de chrome Windows/macOS brut. Tout est dessiné par l'IDE lui-même, à l'image de ce que font JetBrains Toolbox, Unity Hub, ou l'écran de démarrage de Visual Studio.

Son rôle est de permettre :
- Ouvrir un workspace récent en un clic
- Naviguer dans le système de fichiers pour charger un workspace ou dossier
- Créer un nouveau workspace Jenga avec ses projets de démarrage
- Configurer les toolchains, plateformes, et configurations du workspace
- Cloner un dépôt Git et l'ouvrir directement
- Accéder aux paramètres globaux de l'IDE sans ouvrir l'éditeur

### Dimensions & Comportement de la Fenêtre

- **Taille par défaut** : 960 × 620 px, centrée à l'écran
- **Redimensionnable** : oui, avec minimum 760 × 500 px
- **Décoration** : barre de titre custom (pas la barre système), boutons Fermer / Réduire / Maximiser custom
- **Sans bordure système** (frameless window) — l'IDE dessine sa propre barre de titre
- **Toujours au premier plan** au démarrage (puis peut être mise en arrière-plan)
- **Drag** de la barre de titre custom → déplacer la fenêtre
- Double clic sur la barre de titre → maximiser / restaurer

### Carte de Workspace Récent

Chaque entrée de workspace récent est une carte cliquable :

```
┌──────────────────────────────────────────────────────────────────┐
│  🧱  MonMoteur                                    [⭐] [⋯]       │
│      /home/user/projects/MonMoteur                               │
│      Langages: C++20  ·  Configs: Debug, Release                │
│      Plateformes: Windows, Linux, Android, Web                  │
│      Projets: Renderer, Physics, Audio, UI, Main (5)            │
│      Dernier build: ✅ Debug  ·  Modifié: il y a 2h             │
└──────────────────────────────────────────────────────────────────┘
```

Éléments de la carte :
- **Icône** : emoji ou icône détectée depuis le workspace (configurable)
- **Nom** du workspace (le `workspace("nom")` dans le `.jenga`)
- **Chemin** complet (cliquable → révèle dans l'explorateur de fichiers)
- **Langages & Dialectes** détectés
- **Configurations** (Debug, Release, Custom...)
- **Plateformes** cibles
- **Nombre de projets** et leurs noms
- **Dernier statut de build** (✅ succès, ❌ échec, ⚠ partiel)
- **Date de dernière modification**
- `[⭐]` → Épingler en favori (apparaît tout en haut, hors groupes de dates)
- `[⋯]` → Menu contextuel

#### Menu Contextuel sur une Carte

```
  ▶ Ouvrir
  ▶ Ouvrir dans une nouvelle fenêtre
  ─────────────────────
  ▶ Ouvrir dans le terminal
  ▶ Révéler dans l'explorateur
  ─────────────────────
  ▶ Épingler / Désépingler
  ▶ Renommer dans les récents
  ─────────────────────
  ▶ Copier le chemin
  ▶ Supprimer des récents
```

### Groupes des Récents

Les workspaces sont groupés par date :
- **Épinglés** ⭐ (toujours en premier)
- **Aujourd'hui**
- **Cette semaine**
- **Ce mois**
- **Plus anciens**

Chaque groupe est pliable (clic sur le titre).

### Barre de Filtre

```
🔍 [Filtrer les récents...                    ]  [C++ ▼]  [Windows ▼]
```

- Filtre par nom de workspace en temps réel
- Filtre par langage (C++, Python, Rust, Go, JavaScript…)
- Filtre par plateforme (Windows, Linux, Android, Web…)

## 3. Navigateur de Fichiers & Dossiers (Custom File Browser)

### Description

Boîte de dialogue de navigation custom — dessinée entièrement par l'IDE, sans utiliser le dialog système natif (`GetOpenFileName`, `NSOpenPanel`, etc.). L'objectif est une expérience cohérente sur toutes les plateformes, avec des fonctionnalités supplémentaires (prévisualisation, favoris, détection de workspaces Jenga).

### Activation

- Clic sur **Ouvrir un Workspace** → mode "chercher un `.jenga`"
- Clic sur **Ouvrir un Dossier** → mode "chercher un dossier"
- Clic sur n'importe quel champ de chemin dans les wizards → s'ouvre en inline ou fenêtre modale

### Panneau Gauche — Favoris & Appareils

- **Favoris** : raccourcis permanents vers les dossiers fréquemment utilisés
  - Drag d'un dossier depuis la zone principale → ajouter aux favoris
  - Clic droit sur un favori → Renommer, Supprimer
- **Projets** : dossiers détectés comme contenant des workspaces Jenga
- **Appareils** : disques, lecteurs réseau, montages
- Clic sur un item → naviguer vers ce dossier

### Zone Principale — Liste des Fichiers

Affichage en liste (ou grille, basculable) :

| Colonne | Description |
|---------|-------------|
| Icône + Nom | 📁 dossier, 🧱 workspace `.jenga`, 📄 fichier autre |
| Type | "Workspace Jenga", "Dossier", "Fichier" |
| Date de modification | |
| Taille | (pour les fichiers) |
| Projets | (pour les `.jenga` : nombre de projets détectés) |

- Les fichiers `.jenga` sont mis en avant (icône spéciale, fond légèrement coloré)
- Les dossiers contenant un `.jenga` affichent un sous-item visible immédiatement
- Double clic sur un dossier → naviguer dedans
- Double clic sur un `.jenga` → le sélectionner et valider
- Clic sur un `.jenga` → prévisualisation dans un tooltip

### Barre de Navigation (Breadcrumb)

```
[←] [→] [↑] [🏠]   📍  /  home  /  user  /  projects  /  MonMoteur
```

- `←` / `→` → historique de navigation
- `↑` → dossier parent
- `🏠` → dossier home
- Chaque segment du breadcrumb est cliquable (naviguer vers ce niveau)
- Clic sur `📍` → passer en mode saisie libre du chemin
- `Ctrl+L` → activer la saisie libre du chemin

### Mode Saisie Libre

```

```

- Autocomplétion du chemin (TAB pour compléter)
- Suggestions de dossiers en dropdown
- `Enter` pour valider, `Escape` pour annuler

- `[☑ Workspaces .jenga]` — afficher les fichiers `.jenga`
- `[☑ Dossiers]` — afficher les dossiers
- `[☐ Fichiers cachés]` — afficher les fichiers/dossiers commençant par `.`
- `[☐ Tous les fichiers]` — afficher tous les types
- Vue : `[≡ Liste]` `[⊞ Grille]`

### Vérification au Démarrage du Launcher

À chaque ouverture du Launcher, les vérifications suivantes sont effectuées silencieusement :
1. Jenga est-il dans le PATH ? Version correcte ?
2. Python 3.8+ disponible ?
3. Git disponible ?
4. Toolchains configurés dans le registre — tous toujours accessibles ?
5. Workspaces récents — leurs fichiers `.jenga` existent-ils encore ?
6. Mises à jour de l'IDE disponibles (si connexion réseau)

Résultats affichés dans la barre de statut basse du Launcher.

## Annexe B — Flux de Travail Complets

### Créer et Lancer un Nouveau Jeu Multi-Plateforme

1. Launcher → **Nouveau Workspace** (`Ctrl+N`)
2. Étape 1 : Nom = `MonJeu`, Emplacement = `/projects/MonJeu`, Configs = Debug + Release
3. Étape 2 : Ajouter projet `MonJeu` (windowedapp, C++20), ajouter projet `GameLib` (staticlib)
4. Étape 3 : Sélectionner clang-mingw (Win), zig-linux-x64 (Linux), android-ndk (Android), emscripten (Web)
5. Étape 4 : Cocher Windows, Linux, Android, Web · ABIs = arm64-v8a + x86_64
6. Étape 5 : Vérifier l'aperçu `.jenga` → **Créer le Workspace**
7. L'IDE s'ouvre avec le workspace → `Ctrl+Shift+B` pour build → `F5` pour run

### Reprendre un Projet Existant

1. Launcher s'ouvre → voir `MonJeu` dans les récents
2. Clic simple → ouvre directement dans l'IDE (session précédente restaurée)

### Rejoindre un Projet d'Équipe

1. Launcher → **Cloner depuis Git** (`Ctrl+G`)
2. URL : `https://github.com/monequipe/monjeu.git`
3. Destination : `/projects/`
4. Branche : `develop`
5. **Cloner** → scan auto du `.jenga` → ouverture dans l'IDE
